API

Application Programmig Interface, ou Interface de programação de Aplicativos é um conjunto de rotinas e padrões estabelecidos por um software para utilização das suas funcionalidades por outros programas e aplicações que não têm interesse nos detalhes da sua implementação mas usufruem dos seus serviços e dados de uma forma padronizada e acessível em linguagam de máquina.

Durante a fase de testes e implmentação, a api do Votaio não irá requerer nenhum tipo de autenticação para o acesso aos dados. Isso estará sujeito a mudanças em benefício da estabilidade do sistema assim que tivermos demanda sufuciente para fornecer de forma rápida e com confiabilidade os dados públicos que nos comprometermos a fornecer, com um nível excelente de confiabilidade e qualidade.

A proposta do votaio.com é ser uma fonte de dados públicos por meio de uma interface REST que aceita os verbos de leitura do protocolo HTTP e responde em XML, Json ou texto puro de forma consistente, virtualmente tudo que há disponível de dados sobre o cotidiano político do país.

Proposições

Consultas poposições

Uma proposição legislativa é descrita por

  • identificador único (id_sileg)
  • descrição composta de um código alfabético de 3 letras, um numero sequencial e o ano
  • despacho, ou ordem que acompanha a proposição
  • ementa
  • orgão propositor
  • apresentacao que informa a data da proposição no formato "YYYY-MM-DD"

O endereço votaio.com/proposicoes/<numero_da_proposicao> devolve o conteúdo da proposição e seus andamentos passados, em ordem decrescente de data.

Para simples listagem e descrição de proposição, os dados estão acessíveis por:

  • http://localhost:3000/proposicoes/<id igual ao da camara> para retorno html ou
  • http://localhost:3000/proposicoes<id igual ao da camara>.xml

Busca

Para busca aproximada você pode contar com:

  • http://localhost:3000/proposicoes/busca/?q=<chave de pesquisa> para retorno html
  • http://localhost:3000/proposicoes/busca/[xml | rss | json]/<chave de pesquisa> para retorno formatado

Os resultados de busca aproximada levam em conta a presença e repetição de termos nos seguintes campos, por ordem de importância:

  1. autor
  2. descrição
  3. ementa
  4. código

Ou seja uma proposição do deputado Emilio Batista é mostrada antes de uma ocorrência tratando sobre o nome de uma rua chamada Emilio Batista.

Os termos da pesquisa podem ser pesquisados com operadores de exclusão, inclusão, e associações do tipo E e OU lógicas. Por padrão, termos separados por espaços são entendidos como concatenados por operadores do tipo OU lógico.
